# 04. 제품 · UX · 데이터

> 기획 고도에서 "무엇을/왜". 구현(어떻게)은 팀 합류 후 별도 스펙.

## 1. 검사 흐름 (Question Flow)

### 1-1. 기본 원칙
- **진행률 바(progress bar) 필수**: 다단계 이탈 방지. 남은 문항/단계 상시 노출.
- **점진적 노출**: 한 화면에 소수 문항 → 인지 부담↓, 완주율↑.
- 무가입 응시 가능(진입장벽↓), 결과 저장 시 계정 유도.
- 페이싱: word-journey처럼 "서두르지 말라"는 성찰형 톤도 평가형에 잘 맞음.

### 1-2. (#) "단계별 결과 추정치" — 검토만, 지금 구현 안 함 ⚠️
- **아이디어**: 진행 중 "지금까지의 잠정 유형" 미리보기 → 몰입·완주 동기.
- **리스크**: 중간 추정치를 보면 응답자가 그 결과에 맞춰 답을 바꿔 **검사 결과를 오염**(자기충족적 편향)시킬 수 있음.
- **결정**: **Phase 1에서 구현하지 않는다.** 아래 옵션으로만 보관.
  - *옵션 A*: 유형이 아니라 **"완성도/신뢰도 게이지"**만 보여줌(예: "80% 파악됨") — 결과 내용은 숨겨 편향 최소화.
  - *옵션 B*: 추정 유형은 **검사 완전 종료 후에만** 공개(진행 중 비공개).
  - *옵션 C*: A/B 테스트로 오염 여부를 실측한 뒤에만 도입 판단.
- → 채택 여부는 심리학 담당(B)의 편향 검토 + 데이터 담당(A)의 A/B 실험 후 결정.

## 2. (&) 행동 데이터 수집 — "어떻게 답했는가"도 데이터

word-journey처럼 **결과값뿐 아니라 응답 과정**을 수집해 결과 품질을 높인다.

### 2-1. 수집 후보 신호
- 문항별 **소요 시간(반응 지연)**
- **선택 수정 횟수/패턴**(답을 바꿨는가, 몇 번)
- 슬라이더 이동 궤적(망설임), 되돌아가기, 이탈·재개
- 전체 소요 시간, 문항 순서 효과

### 2-2. 활용 (학술 근거는 [03](03-research-plan.md) A-3)
- **1순위 — 성실도/품질 필터**: 너무 빠른 일괄응답 등 careless responding 탐지 → 신뢰도 낮은 결과에 경고/재검사 권유.
- **2순위 — 확신도·양가성 신호**: 오래 망설이거나 자주 바꾼 문항 = 자기인식이 모호한 영역 → 결과 해석에 "당신이 갈등하는 지점" 같은 서사로 활용.
- **3순위(신중) — 특성 추론 보정**: 반응시간을 유형 판정에 직접 반영하는 것은 **자체 검증(타당도) 후에만**. 초기엔 수집·관찰만.

### 2-3. 원칙
- **수집 인프라는 Phase 1부터** 심되, **결과 반영은 검증 후 단계적**으로.
- 개인정보·동의: 무엇을 왜 수집하는지 고지, 최소수집·익명화. (→ 조사 백로그 B-2)

## 3. 결과 페이지 (제품의 핵심)

- **일부 무료 공개 + 공유 가능**(확정 방향). 결과를 가두면 바이럴 죽음.
- **유형 캐릭터화**: 별명·컬러·일러스트 → "나를 표현" 욕구.
- 심층 분석·변화추적·케어 연결은 유료/확장(Phase 3).
- **화살표·발달수준 기반 "현재 심리 건강 상태"**는 Phase 3, 강한 윤리 안전장치와 함께 → [05](05-psychology-foundation.md).

## 4. (3) 공유 카드 + 캐릭터 — 1순위 바이럴 기능

- **한눈에 이해**: 유형명 + 핵심 키워드 3~5개 + 대표 비주얼. 텍스트 최소.
- **귀여운 캐릭터**: 유형마다 고유 캐릭터(브랜드 자산).
- **성격에 맞는 애니메이션 캐릭터 매칭**: "당신과 같은 유형의 캐릭터" 함께 노출 → 공감·공유 동력.
  - ⚠️ **저작권 주의**: 실제 애니 캐릭터 사용은 라이선스 이슈 → (a) 자체 캐릭터로 대체, (b) 라이선스 확보, (c) "닮은 원형(아키타입)"으로 우회 중 택. (→ 조사 백로그 B-3)
- **기술 요건**: 카톡/인스타 스토리 규격 **동적 이미지 자동 생성 + OG 태그**. (조사 B-2)
- **재공유 루프**: "친구와 궁합", "우리 유형 비교"(Phase 2).

## 5. (%) 다양한 테스트 운영 (ingan.ai 참고)

- Phase 1은 **대표 검사 1종 집중**(신뢰). 다양화는 Phase 2.
- 다양화의 두 축:
  1. **검사 종류 확장**: 에니어그램, MBTI, 애착유형, 연애스타일, 퍼스널컬러 등.
  2. **한 결과의 다면 제시**: ingan.ai처럼 하나의 데이터를 성격·관계·성장·위험요인 등 **여러 차원 리포트**로 쪼개기.
- 테스트 간 **결과 연결**(교차 인사이트)이 deepinme만의 깊이가 될 수 있음(예: 에니어그램 × 애착유형).
- 콘텐츠 파이프라인: 푸망식으로 신규 테스트를 정기 출시(C 주도, B 감수).
