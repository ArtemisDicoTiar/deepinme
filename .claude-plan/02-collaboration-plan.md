# 02. 분업 계획 (3인 협업)

3인 팀. 각자 뚜렷한 전문성을 가지되, 이 제품은 **세 영역이 결과물 한 곳(검사 → 결과 → 공유/케어)에서 만나기** 때문에 인터페이스(주고받는 산출물)를 명확히 정의하는 것이 핵심.

## 1. 역할 정의

### 👤 A. 기술 개발자 & 데이터 분석
- 웹/앱 개발, 인프라, 배포
- 검사 채점 로직 구현 (심리학 담당의 알고리즘을 코드화)
- **행동 데이터(반응시간·수정 이력) 수집 파이프라인** 및 저장·분석 → [04](04-product-ux-data.md)
- 데이터 분석: 문항 타당도 통계, 완주율·공유율 등 KPI 대시보드
- 개인정보·보안 처리

### 👤 B. 심리학 전공자
- **검사 프레임워크·문항·채점 기준의 학술적 타당성** 담당 (제품 신뢰의 근간)
- 에니어그램 화살표 이론·발달 수준 등 이론 반영 설계 → [05](05-psychology-foundation.md)
- 결과 해석 텍스트 감수, 이론 출처 정리
- **심리 케어 연결의 윤리·안전장치** 설계 ("진단 아님" 경계, 위기 대응 문구)
- 행동 데이터의 심리학적 해석 가능성 판단 (무엇이 유의미한 신호인지)

### 👤 C. 마케팅 / UIUX / 브랜딩
- 브랜드·톤·캐릭터 세계관 (귀여운 캐릭터/애니 캐릭터 매칭 컨셉)
- 검사 흐름 UX, 결과·**공유 카드 디자인**(한눈에 이해) → [04](04-product-ux-data.md)
- 그로스: 바이럴 루프, SEO 콘텐츠, 신규 테스트 기획
- 유입·전환 퍼널 설계

## 2. 책임 매트릭스 (RACI 요약)
`R=실행 A=최종책임 C=자문 I=공유`

| 활동 | A.기술·데이터 | B.심리학 | C.마케팅·UX |
|---|---|---|---|
| 검사 문항·채점 설계 | C | **A/R** | I |
| 채점 로직 구현 | **A/R** | C | I |
| 결과 해석 텍스트 | I | **A/R** | C(문체) |
| 결과/공유 카드 UX·디자인 | C | C | **A/R** |
| 캐릭터·브랜딩 | I | C(성격정합) | **A/R** |
| 행동데이터 수집·분석 | **A/R** | C(해석) | I |
| SEO·유형별 페이지 | R(구현) | C(내용) | **A/R** |
| 심리 케어 연결·윤리 | R(연동) | **A/R** | C(전달) |
| 로드맵·범위 결정 | C | C | C · *공동* |

## 3. 협업 프로세스

### 인터페이스(산출물)로 연결 — "핸드오프 계약"
경계에서 오가는 산출물을 문서로 고정하면 3인이 병렬로 일할 수 있다.
- **B → A**: 문항 목록 + 채점 규칙 명세(스프레드시트/문서) → A가 구현
- **B → C**: 유형별 해석 원문 + 이론 근거 → C가 톤·비주얼로 가공
- **C → A**: 결과/공유 카드 디자인 스펙 → A가 구현
- **A → B/C**: 수집 데이터 스키마 + 분석 리포트 → 해석·기획에 활용

### 작업 주기 (제안)
- **주 1회 동기화 회의**: 로드맵·블로커·의사결정 (30~60분)
- **비동기 기본**: 이슈 트래커 + 문서. 소규모라 과한 프로세스 지양.
- **의사결정 원칙**: 담당 영역은 담당자 최종결정(A열), 교차 영역은 주 회의에서 합의. 막히면 "Phase 1 최소범위" 원칙으로 판단.

### 도구 (초안 — 팀 합의 필요)
- 코드/이슈: Git 저장소 + 이슈 트래커
- 문서/기획: 공용 문서 도구 (이 `.claude-plan`을 단일 출처로 유지)
- 디자인: Figma 등
- 검사 명세·데이터: 스프레드시트

## 4. 병렬 착수 가능 항목 (Phase 1 초기, 서로 안 막힘)
- **B**: Phase 1 검사 선정 + 문항·채점 초안, 이론 출처 정리
- **C**: 브랜드·캐릭터 컨셉, 검사 흐름·결과 화면 와이어프레임
- **A**: 기술 스택 선정, 저장소·배포 골격, 행동데이터 수집 PoC

> 세 사람이 첫 주에 각각 위 항목을 시작하고, 주 회의에서 인터페이스(문항 명세·디자인 스펙·데이터 스키마)를 맞추면 Phase 1이 빠르게 수렴한다.

## 5. 협업 리스크
- **B의 병목**: 검사·해석·윤리가 모두 B에 몰림 → 우선순위를 검사 1종에 집중, 나머지는 Phase 2로 미룸.
- **경계 모호(캐릭터 성격 정합)**: C가 만든 캐릭터가 심리 유형과 안 맞을 수 있음 → B가 "성격 정합성"만 가볍게 감수.
- **소통 과부하 vs 과소**: 주 1회 + 비동기로 균형.
