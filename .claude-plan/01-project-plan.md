# 01. 프로젝트 계획

## 1. 제품 비전

**deepinme**는 심리 검사를 "재미로 한 번 하고 마는 것"에서 **"나를 이해하고, 필요하면 도움으로 이어지는 여정"**으로 끌어올린다.

- 진입: 신뢰할 수 있는 무료 심리 검사(에니어그램·MBTI 등)
- 심화: 결과의 심층 분석, 시간에 따른 변화, 관계·궁합
- 연결: 필요 시 전문 심리 상담/케어로 연결 (ingan.ai가 보여준 "AI 심리분석 → 상담" 흐름 참고)

README의 철학과 정합: 자기설명력(self-explanation)을 높여 상호이해를 돕는다.

## 2. 단계별 로드맵

> 순서 원칙: **신뢰(평가형)를 먼저 쌓고, 그 위에 재미(엔터테인먼트)와 케어를 얹는다.** 재미부터 시작하면 "그저 그런 테스트 사이트"로 포지셔닝되어 케어 연결의 신뢰가 무너진다.

### Phase 1 — 평가형 코어 (신뢰 구축)
**목표**: "정확하고 믿을 만하다"는 인상. 1개 검사를 깊게.
- 대표 검사 1종을 정식으로 구현 (에니어그램 **또는** MBTI 중 택1 — 심리학 담당이 타당도·문항 확정)
- 이론 출처 명시된 결과 리포트 (무료 공개 범위 확정)
- 유형별 랜딩페이지(SEO 자산)
- 결과 공유 카드 + 캐릭터 매칭 (엔터테인먼트로 가는 다리)
- 행동 데이터 수집 **인프라만** 우선 심음(수집하되 결과 반영은 검증 후) → [04](04-product-ux-data.md)

### Phase 2 — 엔터테인먼트 확장 (유입·바이럴)
**목표**: 트래픽·리텐션. 다양한 테스트로 재방문.
- 짧은 테스트 시리즈 지속 출시 (푸망식)
- 다양한 테스트 포트폴리오 (ingan.ai식 다면성) → [04](04-product-ux-data.md)
- 궁합/친구비교 등 관계형 기능(재공유 루프)
- 캐릭터 세계관 강화(브랜딩 자산)

### Phase 3 — 심화·케어 연결 (수익·사회적 가치)
**목표**: 자기이해 → 성장/케어. 차별화의 핵심.
- 심층 리포트(유료, freemium)
- **화살표 이론·발달 수준 기반 "현재 심리 건강 상태"** 제시 → [05](05-psychology-foundation.md)
- 재검사 히스토리·변화 추적
- 전문 상담/심리 케어 파트너 연결 (윤리·법적 검토 필수 → [05](05-psychology-foundation.md))

## 3. 범위 (Scope) — 초기 원칙

**In (Phase 1)**: 검사 1종, 결과·공유·캐릭터, 유형별 SEO, 행동데이터 수집 인프라, 기본 계정/결과저장.
**Later**: 다양한 테스트, 궁합, 유료 리포트, 케어 연결, 변화추적.
**Out(당분간 배제)**: SNS 데이터 크롤링 기반 분석(ingan.ai식)은 개인정보·동의 리스크가 커서 초기 배제. 자체 설문+행동데이터로 시작.

## 4. 성공 지표 (KPI 초안)
- 신뢰: 검사 **완주율**, 결과 만족/정확도 자기평가
- 바이럴: **공유율**, 공유 경유 신규 유입(K-factor)
- SEO: 유형별 페이지 유입, 검색 노출
- 리텐션: 재방문·재검사율, 신규 테스트 소진율
- 전환: 무료→유료 리포트, 케어 연결 클릭·전환

## 5. 주요 리스크 & 대응
| 리스크 | 영향 | 대응 |
|---|---|---|
| 검사 타당도 부족 → 신뢰 붕괴 | 케어 연결 전체 무너짐 | 심리학 담당이 문항·채점 검증, 출처 명시 |
| 심리 케어 연결의 법적·윤리 문제 | 서비스 중단 위험 | "진단 아님" 고지, 전문가 감수, 유자격 파트너 → [05](05-psychology-foundation.md) |
| 행동데이터/개인정보 처리 | 규제·신뢰 | 최소수집·동의·익명화, 개인정보처리방침 → [04](04-product-ux-data.md) |
| 평가형이 지루해 이탈 | 유입 실패 | 캐릭터·공유·페이싱으로 재미 보강 |
| 3인 소규모 리소스 한계 | 범위 과다 | Phase 1 최소범위 엄수, 검사 1종 집중 |
